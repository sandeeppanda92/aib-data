Gaia Testnet Roadmap

Feature Showcase of the Gaia Testnets in Cosmos/Tendermint

Gaia Testnet Roadmap

This testnet roadmap gives you an idea about where we are in terms of testing and how much more there is to go. In a separate post, you’ll be given a quickstart guide about how to connect to the latest release of the gaiatestnets.

Gaia, Defined

Gaia is the ancestral mother of all life.

The gaia software is what will power the Cosmos Hub, a multi-asset custodial token hub used to enforce global invariants between hundreds to thousands of independent zones and pre-existing blockchains (i.e. Bitcoin, Ethereum). It allows users to move tokens seamlessly between, for instance, a UTXO-based payment zone and an account-based smart contract zone (i.e. Ethermint).

This is a loose showcase of the features in each testnet but it is not an explicit roadmap. Things may change depending on what obstacles we encounter along the way.

Testnet Roadmap

The purpose of each public testnet is to iteratively test the Tendermint Core & Cosmos software stacks. It also gets validators and users familiar with running the software before going live on the main network.

gaia-1—released 2017

  • Dynamic validator set

gaia-2—current release

  • Basic delegation. Enables simple delegation functionality such as delegating tokens to a validator, which gives your voting power to the validator. Enables unbonding, which takes away your voting power from the validator.
  • UI integration for bonding and unbonding
  • PEX reactor. Enables dynamic peer discovery by default.

gaia-3—February release with a focus on the p2p layer

  • Seed mode. There’s a new “seed mode” for seeds to crawl the network and get a large list of peers to provide good first connections for new peers.
  • Sentry node architecture. There are “persistent peers” such as sentry nodes or trusted nodes enabled for testing.
  • Peer IDs. We use peer IDs to prevent man-in-the-middle attacks and have much better handling in the PEX reactor to prevent abuse.
  • Bug fixes: Lots of bug fixes from things people found on gaia-2!

Upcoming Features

  • HSMs. Enables validators to use HSMs for validation.
  • Rewards, fees and provisions.
  • Cosmos-SDK integration.
  • Unbonding period.
