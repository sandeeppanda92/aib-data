How to Connect to the Latest Testnet

Install Gaia

For now we are only supporting installing from source, since it gives you the ability to audit the code you compile. Version 1.9.2 of Go is required to run a node in Gaia. Follow the installation instructions to install the most recent version of Go.

Now rungo get github.com/cosmos/gaia to download the latest release of Gaia. Please disregard can't load package: ... . This is normal and has something to do with how to install software written in Go.

Switch into the correct directory with the following command cd $GOPATH/src/github.com/cosmos/gaia and run make get_vendor_deps to install the required dependencies. Finally install everything by running make install.

Please test whether running gaia version in a terminal produces v0.5.0. If it doesn’t, please go back through the steps above. There is no point in continuing. If you can’t fix it, please ask us on Riot.chat.

Generate Testnet Keys

Private keys are the means by which users authorize spending from their accounts. The Gaia software allows you to generate new private keys.

Friendly reminder: If you don’t hold the private key to your account, you don’t control the tokens in that account. As always, losing your private key to someone else means that they then have control over the tokens in your account.

WARNING: DO NOT INPUT THE 12 WORDS YOU GENERATED DURING THE FUNDRAISER. THERE IS NO REASON TO TAKE THEM OUT OF YOUR SAFETY DEPOSIT BOX UNTIL THE NETWORK IS LIVE.

To generate a new key, run gaia client keys new defaultwhere the name is set to default. You could give it any name you want. Please write down the recovery phrase. This way you don’t have to generate a new address for every testnet.

You can test whether it worked correctly or not by running gaia client keys list. Furthermore, in a future testnet, you will be able to restore your private key by running gaia client keys recover and entering the seed phrase that you saved in the previous paragraph.

Download the Genesis Files

Genesis files are the initial files with preset parameters like block time and initial accounts.

To download genesis files, run git clone https://github.com/tendermint/testnets $HOME/testnets .

Connect to the Network

Now it is time to connect to the gaia-2 testnet for the first time. Run gaia node start --home=$HOME/testnets/gaia-2/gaia. Now you should happily see blocks streaming by as your local node synchronizes with the global public testnet.

Install the Cosmos UI

Synchronisation takes a little while and during that time you might as well install the new Cosmos UI. You can also use the gaia cli as described here to interact with gaia-2, but the UI looks way prettier. You need Node installed and we highly recommend you to follow the NVM instructions. Please check that everything is correctly installed by running node version and npm version. If you see something likeCommand not found, it didn’t work and you have to solve the installation problems first. Join us in Riot.chat if you run into problems.

Once this prerequisite is completed, you’ll need to download the source code for the UI by running git clone https://github.com/cosmos/cosmos-ui $HOME/cosmos-ui. Afterwards please change directory by running cd $HOME/cosmos-ui. Please run yarn to install all the required dependencies. The installation instructions for yarn can be found here.

Once everything is installed, you can run COSMOS_NODE=tcp://localhost:46657 npm run testnet gaia-2. You might have to run the previous command twice.

Now you should see a super sleek UI and will be able to make transactions.

Please wait until the full node has synced with the testnet. This may take a while.

Alternatively, a dead simpler way to run the Cosmos-UI is to go to the binary release, download the zip file, extract the zip file, then double click the ATOM icon. Don’t forget to save your mnemonic.

  • Cosmos-UI v0.4.0 Binary: https://github.com/cosmos/cosmos-ui/releases/tag/0.4.0

The Linux and Mac binaries of the Cosmos-UI is now deterministic. Additionally, seed input into the application has been disabled, so it’s safer relative to the previous release.

Just for Fun

Once the network is synchronized, you can start testing the UI and CLI. The UI is self-explanatory. Try and send some transactions or request funds from others on Riot.chat.

The CLI is for the adventurous. Run gaia client --help to find all the available commands.

Troubleshooting

In case you run into unexpected behaviour, see crashes or other errors, please write us on Riot.chat or submit an issue on either the CLI repo or the UI repo.
