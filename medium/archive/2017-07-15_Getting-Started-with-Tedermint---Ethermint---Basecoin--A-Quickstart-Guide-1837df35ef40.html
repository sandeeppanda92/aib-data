Getting Started with Tendermint Quickstart Guide @ DLT Jam Session

After Hyperledger’s event named Blockchain Ecosystem there was a workshop hosted by Simply Business, part of the series DLT Jam Session, characterised for being technical and hands-on. On this particular day, Tendermint and R3 provided demos of their products on a live-coding basis. I attended Termdermint’s session, but I will try get the material of R3’s session ASAP!

To me, it was a great way of understanding and trying Tendermint’s products. During keynotes or explanations, it can a bit hard to process what Tendermint, Cosmos, Ethermint, Basecoin, etc. are (so many names and sometimes so abstract!). But listening to one of their developers talk about them while typing commands and running the live demo can make it a lot more tangible, easy to understand, and to use. The workshop was live streamed and the record was provided by Makoto from Simply Business. Still, I decided to write this post with the steps taken and commands that were run (on OSX only, sorry for Linux/Windows users!). Hope you find it useful!

Getting Started with Tendermint

First, you will find all the binaries here. After the page is loaded, download tendermint, basecoin and ethermint, save the files in an easy to find place or just in your Downloads folder. Once the binaries have been downloaded, unzip the files (when un-zipable) to the same directory.

Open your terminal and type the command below to move every binary to /usr/local/bin. Change {binary name} for each of the binary file names: tendermint, basecoin, basecli (comes in the same zip as basecoin), and ethermint-darwin-10.6-amd64 (YUP, these package naming inconsistencies are temporal).

$ mv {binary name} /usr/local/bin

Once all the binaries have been moved, type the following commands to check that they are correctly placed. If the commands output something, not an error message, then all set to go!

(Yup, more inconsistencies (┛◉Д◉)┛彡┻━┻).

$ tendermint version
$ basecoin --version
$ ethermint version

The first command is the following, to set local variables we need to create this folder in the specified directory. If you have used Tendermints’ testnet(s) before, you might be overriding them.

$ TESTNET_FILES=~/.cosmos-testnets/files

Then we need to download some init files, the command below and once you downloaded them, navigate to the folder:

$ git clone https://github.com/tendermint/testnets $TESTNET_FILES
$ cd $TESTNET_FILES

Next, running the following command, which sets up some environment variables that are local to the current shell session.

Note: If you’re running this again later, it will ask if you want to override it, just type n for everything in that case.

$ source scripts/testnet-setup.sh

The next commands are to initialise the lite client in two different validating nodes. Mercury and Hermes are the names of two data directories of the 7 validating nodes controlled by Tendermint, which are only for the testnets and not the public chain:

$ basecli-hermes init --node=tcp://hermes-node4.testnets.interblock.io:46657 --chain-id hermes
$ basecli-mercury init --node=tcp://mercury-node4.testnets.interblock.io:46657 --chain-id mercury

Now we can create create our private keys. Change demo for whatever name you like and hit enter, it will ask to you generate a password and repeat it. You can also check the list of private keys you have generated so far:

$ basecli-mercury keys new demo
$ basecli-mercury keys list

And check the balance, which will not output anything if you don’t have any coins. Remember to change “3BB9876xx...” for the one you generated before. (And you can send me coins to this address ❤)

$ basecli-mercury query account 3BB9876328E769E658D9E831E65A39E5B1C853C0

Not very pretty, but there’s a command to send coins to other accounts. Replace {key name} for the name that you gave to one of key(s) you generated before. Set amount equal to any amount followed by the name of the coin (in this case, it’s 10 pandas, racist jokes apart), followed by the recipient, and finally the sequence (tricky!) which is the current sequence of your account + 1.

$ basecli-mercury tx send —name={key name} —amount=10pandas 3BB9876328E769E658D9E831E65A39E5B1C853C0 —sequence=18

You can find the current sequence of your account by running:

$ basecli-mercury query account 3BB9876328E769E658D9E831E65A39E5B1C853C0

Finally, you can do the same thing with hermes, yes :

$ basecli-hermes init --node=tcp://mercury-node4.testnets.interblock.io:46657 --chain-id hermes
$ basecli-hermes keys new demo
$ basecli-hermes keys list
$ basecli-hermes query account EA35286890D3132E220B2AB818856672FCA9B600
$ basecli-hermes tx send —name={key name} —amount=10pandas EA35286890D3132E220B2AB818856672FCA9B600 —sequence=19

And that’s it! You have tried the wonders of Tendermint and got a sneak peak into the possibilities of the Cosmos Hub. Check the video below if the steps before were not clear (sorry -.-), it also includes a small explanation of each of these products and some extra demo at the end.
