Testnet Disruption Update

Report

Last night a disruption in testnet Gaia-1 happened. It caused nodes to go down. Cause of the outage is identified and fixed. You can restart your node, it should function properly.

What happened

1- A bug was identified in the way our validator nodes handled logs. This was an operational problem which had nothing to do with Tendermint itself.

2- This caused our validator nodes to go down for 5 minutes.

3- Retries for persistent peers currently only last for about 90 seconds before giving up.

4- Our validator nodes came back online but all other nodes stopped trying to connect to them by then.

5- Peer exchange is not enabled by default so no one had peers to connect to.

Solution

  • Log handling issue on our validator nodes is fixed, so you only need to restart your node and it should work fine.
  • Peer exchange will be enabled by default in the next release. Detailed solution here: https://github.com/tendermint/tendermint/milestone/12
